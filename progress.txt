## Codebase Patterns
- Migrations: Always use `IF NOT EXISTS`. Apply with `npm run migrations:local` / `npm run migrations:staging`.
- CSS: Compile with `npm run build:css` before deploy. Never edit `generated-styles.ts` directly.
- D1 queries: `env.NEXUS_DB.prepare(...).bind(...).all()` / `.first()` / `.run()`
- KV: Sessions (7-day TTL), magic links (15-min TTL), rate limiting, trace counts
- Auth: `nexus_session` HttpOnly cookie. `requireAuth` middleware in `src/middleware/`.
- SQL escaping in .sql files: Use `''` to escape single quotes (NOT `\'` which is shell syntax)
- subscriptions table: Use `ORDER BY rowid DESC LIMIT 1` for latest sub. Production d1_migrations may lag.
- Demo data: Hardcoded in `src/pages/demo.ts` (DEMO_TRACES, DEMO_SPANS). Seed data in D1 via `scripts/seed-demo.sql`.
- Smoke test pattern: `hasTestEndpoints` flag gates bootstrap tests. Production runs with SKIP_AUTH_TESTS=1.
- Pre-existing test failures (not my work): keys.spec empty name, settings.spec non-https URL, mobile webkit missing.
