## Codebase Patterns
- D1 queries on `subscriptions` table: Use `ORDER BY rowid DESC LIMIT 1` NOT `ORDER BY created_at DESC` — the billing route already did this correctly; agents/settings/dashboard had a bug using `created_at` which didn't exist until migration 0008
- Migration tracking: Production D1 migrations were applied manually (outside Wrangler), causing d1_migrations table to be out of sync. Fix: INSERT rows into d1_migrations to mark them as applied, then redeploy
- Smoke test helpers: `hasTestEndpoints` flag gates all bootstrap-dependent tests. Set `SKIP_AUTH_TESTS=1` for production runs. `BASE_URL` env var switches playwright.config.ts between local dev server and remote
- Playwright form submissions: Use `Promise.all([page.waitForLoadState('networkidle'), page.click('button[type="submit"]')])` to wait for POST navigation. HTML5 `required` validation blocks empty submissions — use `page.evaluate()` to remove `required` before testing server-side validation
- Revoke form has `onsubmit="return confirm(...)"` — use REST DELETE endpoint instead of browser form click for cleaner testing
- KV eventual consistency: Don't test browser-redirect behavior after logout against staging/prod — KV writes can take seconds to propagate across edge. Test the API response (redirect status code) instead
